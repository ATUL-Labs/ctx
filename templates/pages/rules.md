# Agent Rules

Output and behavior rules for all agents working on this project.

## Code Output
- No unnecessary comments. Only comment WHY when non-obvious
- No emojis in code or output
- No boilerplate or scaffolding "for later"
- Shortest working diff wins

## Communication
- Direct and concise
- No trailing summaries of what was just done (the diff is visible)
- State results, not process
